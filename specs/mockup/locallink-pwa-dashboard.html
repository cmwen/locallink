MCP tools

These are the four automation primitives exposed to agents and local orchestration flows.

read_infra_config Input: None

Reads .env, docker-compose.yml, and ecosystem.config.js as a structural snapshot.

write_infra_config Input: target_file + content or patch

Safely mutates the selected infrastructure file without flattening comments or formatting.

get_available_port Input: start_from optional

Scans sequentially for the next completely open local port.

execute_task Input: runtime + service_name + action

Runs docker, pm2, or taskfile lifecycle commands and streams the terminal output.

LocalLink constraints

What this Phase 1 shell makes explicit.

Loopback only Phase 1 binds the local UI and the control plane to 127.0.0.1.
External runtime truth Each snapshot re-queries Docker, PM2, and Windows process probes instead of trusting in-memory state.
Unknown when unverifiable Services fall back to Unknown when a runtime manager is unavailable or LocalLink lacks a trustworthy probe.

Activity loop

Short-form events that help developers and AI agents orient quickly.

Service matrix One glance tells you which containers, PM2 workers, and host processes are live.
Log tailing Lifecycle, runtime, and alert events remain readable in a terminal-style pane.

LocalLink Phase 1 is designed as a local-only control plane: no public proxy, no cloud dependency, and no machine-specific absolute paths in the configuration model.
