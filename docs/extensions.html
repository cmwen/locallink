Workspace capabilities

How LocalLink out-of-box extensions work

Extensions declare optional capabilities around the service topology. They do not silently install software, publish ports, or create credentials.

The declaration model

locallink.extensions.yml is the committable capability inventory. LocalLink resolves exposed ports from .env, reports missing required variables as setup work, and displays each extension in Connections. Runtime services remain in docker-compose.yml or locallink.services.yml; private values remain in .env.

An extension marked ready has its declared configuration present. Use the related service card and runtime checks to confirm live connectivity and external authentication.

extension declaration → readiness in dashboard
service declaration   → lifecycle and runtime status
.env                   → local ports, URLs, and secrets

Dashboard

The local control surface. It reads the workspace model, runtime probes, connection declarations, resource state, and saved display preferences. Enabling its preference does not override the configured bind host or launch an undeclared service.

Reverse proxy

Maps stable origins or edge listeners to application ports. Caddy is the default example. LocalLink records the capability and its service dependencies; the Caddyfile remains the routing source of truth.

Tailscale private edge

Provides authenticated network reachability and private HTTPS. Tailscale Serve exposes only the routes explicitly declared in its configuration, and Tailscale ACLs remain the network authorization source. LocalLink never treats edge enablement as permission to make a service public.

Pocket ID

Provides passkey-backed OIDC sessions to internal applications after a user reaches them through Tailscale. It depends on a stable HTTPS issuer, persistent data, and an encryption key. See the private SSO guide.

OpenObserve

Receives telemetry from declared services. LocalLink reports whether the required endpoint and stream variables exist, but application exporters and OpenObserve credentials remain explicit service configuration.

Default safety properties

  • No extension writes secrets into committed files.
  • No network edge is assumed public.
  • Missing required variables produce setup status rather than fake readiness.
  • Extension preferences do not replace Docker, PM2, Taskfile, proxy, or Tailscale configuration.
  • Disabling a dashboard preference does not destructively stop an external runtime.
